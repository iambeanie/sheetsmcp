# Packaging

SheetsMCP should be distributed as self-contained executables so users do not need to install .NET on the target machine.

## Release Targets

Primary runtime identifiers:

- `linux-x64`
- `linux-arm64`
- `osx-x64`
- `osx-arm64`
- `win-x64`

Additional targets may be added when there is a concrete user need.

## Publish Settings

The project should publish with:

- `TargetFramework` set to `net10.0`
- `SelfContained` enabled for release artifacts
- `PublishSingleFile` enabled where compatible
- `PublishTrimmed` disabled by default unless tested with Google client libraries and MCP SDK
- `InvariantGlobalization` disabled unless range/value formatting has been verified

Recommended project properties:

```xml
<PropertyGroup>
  <TargetFramework>net10.0</TargetFramework>
  <OutputType>Exe</OutputType>
  <SelfContained>true</SelfContained>
  <PublishSingleFile>true</PublishSingleFile>
  <PublishTrimmed>false</PublishTrimmed>
</PropertyGroup>
```

## Publish Commands

Use the agent-friendly publishing scripts:

```bash
./scripts/dotnet-publish linux-x64
./scripts/dotnet-publish all
./scripts/dotnet-release
```

Generated output is placed under:

```text
artifacts/publish/<rid>/
artifacts/packages/sheetsmcp-<rid>.tar.gz
artifacts/packages/sheetsmcp-win-x64.zip
artifacts/SHA256SUMS
```

## Artifact Layout

Each release artifact should include:

- the `sheetsmcp` executable or `sheetsmcp.exe`
- `README.md` or a short usage note
- license file when one exists
- checksum file generated by the release process

Do not package service-account JSON credentials with the app.

## Platform Notes

### Linux

Release both x64 and arm64 when possible. The executable may need the executable bit set after extraction:

```bash
chmod +x sheetsmcp
```

### macOS

Release separate Intel and Apple Silicon artifacts. Signing and notarization should be considered before distributing outside local development.

### Windows

Release `win-x64` as the default Windows target. The executable should run from a normal user directory without requiring administrator permissions.

## Verification Checklist

Before publishing a release:

- Run `./scripts/dotnet-verify-all`.
- Publish each runtime identifier.
- Start each published executable and confirm it does not require an installed .NET runtime.
- Validate stdio MCP startup with a local MCP client or protocol smoke test.
- Validate a read call against a shared test spreadsheet when credentials are available.
- Validate a write call against a disposable test spreadsheet when credentials are available.

If Google credentials are not available in CI, keep Google integration checks separate from unit tests and document that they were skipped.
